# Почему запускались множественные процессы Next.js?

## 🔍 Причины

### 1. **Next.js 15 нормальная архитектура** ✅
Next.js 15 **создает несколько процессов** - это нормально:
- `node next dev -p 3003` - основной процесс
- `next-server (v15.5.4)` - worker процесс для обработки запросов

**Это НЕ проблема!** Проблема возникает только когда запущено **несколько экземпляров** `npm run dev`.

### 2. **Двойной запуск** ⚠️
Конфликт возникает, если:
- Вы запустили `npm run dev` в терминале
- **И одновременно** Cursor/IDE автоматически запустил еще один экземпляр
- Или вы запустили второй раз, не остановив первый

### 3. **Cursor автоматический запуск**
Cursor может автоматически запускать dev сервер:
- При открытии проекта
- При работе с агентом (если настроено)
- При изменении файлов (если включен auto-start)

## ✅ Решение

### Проверка перед запуском
Используйте скрипт для проверки:
```bash
./check-dev-server.sh
```

### Безопасный запуск
Вместо обычного `npm run dev` используйте:
```bash
npm run dev:safe
```

Этот скрипт:
1. Проверит, не запущен ли уже сервер
2. Предупредит, если найдены множественные процессы
3. Запустит сервер только если все чисто

### Ручная проверка
Перед запуском всегда проверяйте:
```bash
ps aux | grep "next dev" | grep -v grep
```

Если видите процессы - остановите их:
```bash
./fix-nextjs.sh
```

## 🛡️ Предотвращение

### 1. Всегда проверяйте перед запуском
```bash
# Быстрая проверка
ps aux | grep "next dev" | grep -v grep || echo "Сервер не запущен"
```

### 2. Используйте безопасный запуск
```bash
npm run dev:safe
```

### 3. Настройте Cursor
Если Cursor автоматически запускает сервер:
- Отключите auto-start в настройках Cursor
- Или используйте только ручной запуск через терминал

### 4. Создайте алиас
Добавьте в `~/.zshrc` или `~/.bashrc`:
```bash
alias nextjs-check='ps aux | grep "next dev" | grep -v grep'
alias nextjs-kill='pkill -f "next dev" && pkill -f "next-server"'
```

## 📊 Как отличить нормальные процессы от конфликта?

### ✅ Нормально (1 экземпляр):
```
PID 12345  node next dev -p 3003
PID 12346  next-server (v15.5.4)
```
Это **один набор процессов** - основной + worker.

### ❌ Проблема (2+ экземпляра):
```
PID 12345  node next dev -p 3003
PID 12346  next-server (v15.5.4)
PID 54321  node next dev -p 3003  ← Второй экземпляр!
PID 54322  next-server (v15.5.4) ← Второй worker!
```
Это **два набора процессов** - конфликт!

## 🔧 Автоматическое исправление

Если обнаружены множественные процессы:
```bash
./fix-nextjs.sh
```

Этот скрипт:
1. Остановит все процессы Next.js
2. Очистит кеш
3. Проверит конфигурацию
4. Подготовит к безопасному запуску

## 💡 Рекомендации

1. **Всегда проверяйте** перед запуском `npm run dev`
2. **Используйте** `npm run dev:safe` для автоматической проверки
3. **Останавливайте** сервер через `Ctrl+C` перед повторным запуском
4. **Не запускайте** сервер в нескольких терминалах одновременно
5. **Отключите** auto-start в Cursor, если он мешает

## 🎯 Итог

**Главная причина:** Запуск нескольких экземпляров `npm run dev` одновременно.

**Решение:** 
- Всегда проверяйте перед запуском
- Используйте `npm run dev:safe`
- Останавливайте предыдущий процесс перед новым запуском
