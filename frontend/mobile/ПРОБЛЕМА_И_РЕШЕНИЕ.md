# Проблема: TypeError в createAnimatedComponent

## Описание проблемы

При запуске мобильного приложения на React Native с Expo возникает ошибка:

```
Exception in HostFunction: TypeError: expected dynamic type 'boolean', but had type 'string'
```

**Место возникновения:**
- `createAnimatedComponent.js:107`
- `ReactFabric-dev.js:9200:35`
- `api.js:27` (NativeWind)

## Корневая причина

**Проблема в комбинации:**
- Expo SDK 54 + React Native 0.81 + React 19.1
- **New Architecture включена** (`newArchEnabled: true` в `app.json`)
- NativeWind 4.2.1 в зависимостях, но **Babel-плагин отключен**
- **По всему коду остались `className`**, которые не обрабатываются

### Почему это вызывает ошибку:

1. **New Architecture** в Expo SDK 54 использует Reanimated 4 (вшит в Expo)
2. **NativeWind 4.2.1** несовместим с Reanimated 4 без правильной настройки
3. **`className` без обработки** передаются как пропсы в анимированные компоненты
4. React Native ожидает boolean, но получает строку из `className`

> **Важно:** Expo документация прямо говорит: "If you use Nativewind, you will want to stay on Reanimated v3 for now." Но Expo SDK 54 поставляется с Reanimated 4.

## Следующие шаги

### Вариант А: MVP без NativeWind (рекомендуется)

**Для мобильного MVP важнее стабильность, чем Tailwind-классы.**

1. **Полностью удалить NativeWind:**
   ```bash
   npm uninstall nativewind tailwindcss
   ```

2. **Использовать только StyleSheet:**
   - Создавать `styles = StyleSheet.create({ ... })` в каждом компоненте
   - Переносить все стили из `className` в StyleSheet
   - На MVP можно стилизовать грубо - главное функциональность

3. **Оставить простой Babel config:**
   ```js
   module.exports = function (api) {
     api.cache(true);
     return {
       presets: ["babel-preset-expo"],
     };
   };
   ```

**Преимущества:**
- ✅ Стабильная работа на Expo SDK 54 + New Architecture
- ✅ Нет конфликтов с Reanimated 4
- ✅ Простая отладка

### Вариант Б: Правильная настройка NativeWind для Expo SDK 54

**Если принципиально нужен Tailwind-подход:**

1. Установить совместимые версии:
   ```bash
   npx expo install nativewind@4.2.1 react-native-css-interop@^0.2.1
   npx expo install react-native-reanimated react-native-worklets
   ```

2. Настроить Babel:
   ```js
   module.exports = function (api) {
     api.cache(true);
     return {
       presets: [
         ["babel-preset-expo", { jsxImportSource: "nativewind" }],
         "nativewind/babel",
       ],
       plugins: ["react-native-reanimated/plugin"],
     };
   };
   ```

3. Настроить Metro с `react-native-css-interop`

4. Вернуть `import './global.css'`

5. Аккуратно вернуть `className` в компоненты

**⚠️ Внимание:** Это сложнее и может вызвать новые проблемы. Для MVP не рекомендуется.
