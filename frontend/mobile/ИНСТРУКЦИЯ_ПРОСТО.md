# Мобильное приложение — простая инструкция

Всё ниже делается на **компьютере**, где лежит проект. Телефон — только для установки **Expo Go** и сканирования QR.

---

## 1. Один раз установить

1. **Node.js 20** (на сервере: `nvm install 20` и потом всегда `nvm use 20` перед работой).
2. На **телефоне** — приложение **Expo Go** из магазина (App Store / Google Play).

---

## 2. Один раз настроить файл `.env`

Папка: **`rexten/frontend/mobile/`** (внутри репозитория).

Создай или открой файл **`.env`** и напиши **одну** рабочую строку (без кавычек):

```env
EXPO_PUBLIC_API_BASE_URL=https://api.rexten.live/api
```

Так приложение на телефоне будет ходить в **тот же API**, что и сайт.  
Если откроешь в браузере `https://api.rexten.live/api` и увидишь **404** — это **нормально**: там нет «главной страницы», рабочие адреса вида `/api/...` другие.

---

## 3. Каждый раз: открыть терминал в правильной папке

В терминале **обязательно**:

```bash
cd /путь/к/rexten/frontend/mobile
```

Не запускай Expo из папки `rexten` без `frontend/mobile` — иначе подтянется не тот Expo и будут ошибки.

Проверка версии Node:

```bash
node -v
```

Должно быть **v20.x.x**. Если 18 — сделай `nvm use 20`.

---

## 4. Как запустить — выбери один вариант

### Вариант А — телефон в **той же Wi‑Fi**, что и компьютер

1. Освободи порт (если что-то уже крутилось): закрой старый терминал с Expo или убей процесс на порту 8081.
2. Запуск:

```bash
CI=false npm start
```

3. В терминале появится **QR-код** (или ссылка `exp://...`).
4. Открой **Expo Go** → сканируй QR.

Если QR не видно — в Expo Go выбери ввод URL вручную:

```text
exp://АЙПИ_КОМПЬЮТЕРА:8081
```

Айпи компьютера в Wi‑Fi узнай командой `hostname -I` (берёшь адрес вида **192.168.x.x**, не 172.17 и т.п.).

---

### Вариант Б — телефон **не в той сети** (мобильный интернет, другой город)

**Для Expo Go используй встроенный туннель Expo (ngrok)** — он отдаёт URL, с которым приложение реально соединяется:

```bash
cd /путь/к/rexten/frontend/mobile
CI=false npx expo start --tunnel
```

При запросе согласись поставить **`@expo/ngrok`**. Может попросить **`npx expo login`**. Сканируй QR из терминала.

**Почему был timeout с trycloudflare:** Expo показывает `exp://….trycloudflare.com:8081`. С интернета порт **8081** на этом хосте недоступен (снаружи — в основном **HTTPS на 443**). Expo Go ждёт ответ на **8081** → **The request timed out**. Скрипт `npm run start:trycloudflare` для **обычного браузера** к Metro ещё куда ни шло, для **Expo Go** надёжнее **`--tunnel`**.

---

### Вариант В — свой домен (например metro.rexten.live)

Нужно, чтобы **туннель** (cloudflared на твоей стороне) уже вёл на **http://127.0.0.1:8081**. Это настраивается **в панели Cloudflare / в конфиге туннеля**, не в коде.

Потом в `.env` можно задать:

```env
EXPO_METRO_HOST=metro.rexten.live
```

И запуск:

```bash
npm run start:metro-host
```

---

## 5. Частые ошибки

| Что сделал | Что будет |
|------------|-----------|
| Запуск из корня `rexten`, не из `frontend/mobile` | Ставится чужой Expo, ошибки Node / parseEnv |
| Пишешь `CI=` пустое перед командой | Ошибка Expo про boolean |
| Телефон «извне», а запускаешь только `npm start` без туннеля | Не подключится к Metro |
| В `.env` API на `localhost` с физического телефона | API недоступен — используй `https://api.rexten.live/api` |

---

## 6. Коротко: минимум команд

```bash
cd rexten/frontend/mobile
nvm use 20
CI=false npm start
```

Снаружи дома:

```bash
cd rexten/frontend/mobile
nvm use 20
npm run start:trycloudflare
```

Готово.
