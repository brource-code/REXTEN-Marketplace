# План развития мобильного приложения

Текущая база (навигация, API бизнеса/клиента, авторизация) считается **рабочей отправной точкой**. Дальше развитие идёт **по очереди, экран за экраном**.

## Принцип

- Выстраивать страницы **по аналогии стиля и паттернов веба** (`frontend/src/app/business/…`, общие UI-паттерны маркетплейса): те же смысловые блоки, те же потоки данных, но упрощённая под мобильный форм-фактор реализация (списки, модалки, по шагам).
- **Эталон по UX/копирайту веба** — не дублировать лишнюю сложность там, где достаточно ссылки на веб или одного действия.
- Перед доработкой экрана: кратко свериться с соответствующей страницей веба и с `src/lib/api/business.ts` / клиентским API.

## Порядок работ (гибкий бэклог)

Идём **последовательно**, но порядок внутри блока можно менять по приоритету продукта:

1. **Операционка** — брони, клиенты, расписание: плотнее совпадение с вебом (фильтры, карточки, статусы).
2. **Настройки бизнеса** — услуги, команда, пользователи/роли: формы и валидация ближе к веб-вкладкам.
3. **Объявления и реклама** — списки, черновики, тяжёлые сценарии при необходимости через веб.
4. **Отчёты и биллинг** — фильтры периода, сводки; экспорт при необходимости через веб.
5. **Полировка** — типографика/отступы под дизайн-систему веба, локализация, обработка edge cases.

## Зафиксировано

- База кода пригодна для итераций; новые задачи добавляются в этот файл или в трекер задач команды с отсылкой к **«паритет с вебом по паттернам»**.
