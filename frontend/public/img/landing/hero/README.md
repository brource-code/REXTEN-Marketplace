# Картинки hero лендинга по языкам

Компонент подставляет файлы **по текущей локали** интерфейса (тот же код, что в `useLocale()` и в `messages/*.json`).

## Структура папок (обязательно)

Для **каждого** из пяти языков — своя подпапка, **имя папки = код локали**:

| Локаль UI | Папка |
|-----------|--------|
| English | `en/` |
| Русский | `ru/` |
| Español (México) | `es-MX/` |
| Հայերեն | `hy-AM/` |
| Українська | `uk-UA/` |

## Имена файлов (строго так)

В каждой папке ровно **два** файла:

| Файл | Когда показывается |
|------|---------------------|
| `hero.png` | Светлая тема лендинга |
| `hero-dark.png` | Тёмная тема лендинга |

Другие имена не подхватятся.

## Формат и размер

- Формат: **PNG** (как сейчас в коде).
- Желательно одинаковый размер кадра у всех языков и у пары light/dark, чтобы не прыгала вёрстка (ориентир: те же пропорции, что у текущих макетов).

## Что подставлять в URL вручную не нужно

Путь собирается в коде: `<NEXT_PUBLIC_BASE_PATH>/img/landing/hero/<код локали>/hero.png` (или `hero-dark.png`). Если `NEXT_PUBLIC_BASE_PATH` пустой — то же, что `/img/landing/hero/...`. Достаточно положить файлы в нужные папки под `public/`.

## Файлы в корне `hero/`

Старые `hero.webp`, `hero-dark.webp` и PNG в корне можно **не удалять** — они не используются лендингом, если лежат отдельные наборы в подпапках `en/`, `ru/` и т.д.

## Логика в коде

`frontend/src/app/(public-pages)/landing/utils/landingHeroImages.js` — функция `getLandingHeroImageSrc(locale, dark)`. Неизвестная локаль откатывается на **`en`**.

## Другие блоки лендинга (те же пять локалей)

В `public/img/landing/` лежат копии ассетов по подпапкам **`layouts/<код>/`**, **`features/<код>/`**, **`tech/<код>/`** (имя папки = код локали, как в таблице выше). Сейчас во всех языках — дубликаты одних и тех же файлов; при появлении локализованных скриншотов заменяйте файлы только в нужной папке.

Формат в коде — **только PNG**: в `layouts/<код>/` ожидаются `collapsible.png`, `stacked.png`, `topbar.png`, `frameless.png`, `overlay.png`, `documentation.png`, `documentation-dark.png`; в `features/` — `mobile.png`, `rtl.png`, `rtl-dark.png`; в `tech/` — `<id>.png`.

В коде: `getLandingLayoutsSrc`, `getLandingFeaturesSrc`, `getLandingTechSrc` в том же `landingHeroImages.js`.
