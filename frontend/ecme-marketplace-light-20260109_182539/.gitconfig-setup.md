# Настройка Git для работы с GitHub

## Способ 1: Использование Personal Access Token (рекомендуется)

1. Создайте Personal Access Token на GitHub:
   - Перейдите в Settings > Developer settings > Personal access tokens > Tokens (classic)
   - Нажмите "Generate new token (classic)"
   - Выберите права: `repo` (полный доступ к репозиториям)
   - Скопируйте токен

2. Настройте remote с токеном:
```bash
git remote set-url origin https://YOUR_TOKEN@github.com/brource-code/REXTEN-Marketplace.git
```

3. Отправьте код:
```bash
git push -u origin main
```

## Способ 2: Использование SSH ключа

1. Создайте SSH ключ:
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

2. Скопируйте публичный ключ:
```bash
cat ~/.ssh/id_ed25519.pub
```

3. Добавьте ключ в GitHub:
   - Перейдите в Settings > SSH and GPG keys
   - Нажмите "New SSH key"
   - Вставьте публичный ключ

4. Измените remote на SSH:
```bash
git remote set-url origin git@github.com:brource-code/REXTEN-Marketplace.git
```

5. Отправьте код:
```bash
git push -u origin main
```

## Использование скрипта для автоматических коммитов

После настройки аутентификации вы можете использовать скрипт `.git-commit.sh`:

```bash
# Коммит с автоматическим сообщением
./.git-commit.sh

# Или с вашим сообщением
./.git-commit.sh "Описание изменений"
```

## Настройка Git Credential Helper (для сохранения токена)

```bash
# Для macOS
git config --global credential.helper osxkeychain

# Для Linux
git config --global credential.helper store
```

После первого push с токеном, он будет сохранен и больше не потребуется вводить его каждый раз.
