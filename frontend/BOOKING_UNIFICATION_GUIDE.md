# Руководство по унифицированной системе бронирования

## 📋 Обзор

Система бронирования теперь поддерживает три типа услуг:
- **On-site** (Клиент приходит сам) - для салонов красоты, медицинских клиник, офисов
- **Off-site** (Выезд к клиенту) - для клининга, автосервиса, ремонта
- **Hybrid** (Гибридная) - клиент выбирает вариант

## 🎯 Где посмотреть и настроить

### 1. Настройка типа услуги (Бизнес-админка)

**Путь:** Бизнес-админка → Настройки → Услуги

**Как настроить:**
1. Откройте раздел "Настройки" в бизнес-админке
2. Перейдите на вкладку "Услуги"
3. Нажмите "Добавить услугу" или выберите существующую услугу для редактирования
4. В форме редактирования услуги найдите раздел **"Тип услуги"**
5. Выберите один из вариантов:
   - **Клиент приходит сам (On-site)** - для услуг в салоне/офисе
   - **Выезд к клиенту (Off-site)** - для услуг с выездом
   - **Гибридная (клиент выбирает)** - клиент сам выбирает вариант

**Примеры:**
- Стрижка в салоне → **On-site**
- Клининг на дому → **Off-site**
- Массаж (может быть в салоне или на дому) → **Hybrid**

### 2. Просмотр бронирований (Бизнес-админка)

**Путь:** Бизнес-админка → Дашборд → Последние бронирования

**Что видно:**
- В списке бронирований для выездных услуг отображается индикатор **🚗 Off-site**
- Для обычных услуг (on-site) индикатор не показывается

**Пример отображения:**
```
10:00 — Cleaning — John (🚗 Off-site)
11:30 — Haircut — Anna
```

### 3. Детали бронирования (Бизнес-админка)

**Путь:** Бизнес-админка → Клиенты → [Выбрать клиента] → Вкладка "Бронирования"

**Что видно:**
- Для **off-site** бронирований:
  - Показывается тип исполнения: "Выезд к клиенту"
  - Отображается полный адрес клиента:
    ```
    Адрес:
    123 Main St
    Los Angeles, CA 90001
    ```
- Для **on-site** бронирований адрес не показывается

### 4. Клиентская форма бронирования (Веб)

**Путь:** Публичная страница услуги → Кнопка "Забронировать"

**Как работает:**

#### Для On-site услуг:
- Поля адреса **не показываются**
- Клиент заполняет только: имя, телефон, email, комментарий

#### Для Off-site услуг:
- Поля адреса **обязательны** и показываются автоматически:
  - Адрес (улица и номер)
  - Город
  - Штат
  - Почтовый индекс

#### Для Hybrid услуг:
- Показывается переключатель:
  - **"Я приеду сам"** - поля адреса скрываются
  - **"Приехать ко мне"** - поля адреса становятся обязательными

**Пример формы для Hybrid услуги:**
```
┌─────────────────────────────────┐
│ Тип исполнения *                │
│ ○ Я приеду сам                  │
│ ● Приехать ко мне               │
│                                 │
│ Адрес *                         │
│ [123 Main St          ]         │
│ Город *    Штат *               │
│ [Los Angeles] [CA    ]          │
│ Почтовый индекс *               │
│ [90001              ]           │
└─────────────────────────────────┘
```

### 5. Мобильная форма бронирования

**Путь:** Мобильное приложение → Выбрать услугу → Забронировать

**Как работает:**
- Логика аналогична веб-версии
- Для **hybrid** услуг - переключатель в виде кнопок
- Для **off-site** услуг - поля адреса обязательны

## 🔧 Технические детали

### База данных

**Новые поля:**
- `services.service_type` - тип услуги (onsite/offsite/hybrid)
- `bookings.execution_type` - как будет выполнена услуга (onsite/offsite)
- `booking_locations` - таблица с адресами для выездных бронирований

### API

**Новые поля в запросе создания бронирования:**
```json
{
  "execution_type": "offsite",  // для hybrid услуг обязательно
  "address_line1": "123 Main St",
  "city": "Los Angeles",
  "state": "CA",
  "zip": "90001"
}
```

**Ответ API для бронирований:**
```json
{
  "execution_type": "offsite",
  "location": {
    "address_line1": "123 Main St",
    "city": "Los Angeles",
    "state": "CA",
    "zip": "90001"
  }
}
```

## 📝 Примеры использования

### Пример 1: Салон красоты
1. **Настройка услуги:**
   - Услуга: "Стрижка"
   - Тип: **On-site**
   
2. **Результат:**
   - Клиент при бронировании не видит поля адреса
   - В админке нет индикатора "Off-site"

### Пример 2: Клининговая служба
1. **Настройка услуги:**
   - Услуга: "Уборка квартиры"
   - Тип: **Off-site**
   
2. **Результат:**
   - Клиент **обязательно** заполняет адрес
   - В админке виден индикатор "🚗 Off-site"
   - В деталях бронирования виден адрес клиента

### Пример 3: Массаж (гибрид)
1. **Настройка услуги:**
   - Услуга: "Массаж"
   - Тип: **Hybrid**
   
2. **Результат:**
   - Клиент видит переключатель "Я приеду сам" / "Приехать ко мне"
   - Если выбрал "Приехать ко мне" - заполняет адрес
   - Если выбрал "Я приеду сам" - адрес не нужен

## ⚠️ Важные замечания

1. **Обратная совместимость:**
   - Все существующие услуги автоматически получили тип "On-site"
   - Все существующие бронирования автоматически получили тип "On-site"

2. **Валидация:**
   - Для **off-site** услуг адрес обязателен
   - Для **hybrid** услуг выбор типа исполнения обязателен
   - Система не позволит создать бронирование без обязательных полей

3. **Адрес бизнеса:**
   - Адрес бизнеса (из настроек компании) **не используется** для бронирований
   - Для выездных услуг используется только адрес клиента

## 🐛 Решение проблем

### Проблема: Поля адреса не показываются для off-site услуги
**Решение:** Проверьте, что в настройках услуги выбран тип "Выезд к клиенту (Off-site)"

### Проблема: Не могу создать бронирование без адреса для on-site услуги
**Решение:** Проверьте настройки услуги - возможно, тип установлен как "Off-site" вместо "On-site"

### Проблема: Переключатель не показывается для hybrid услуги
**Решение:** Убедитесь, что в настройках услуги выбран тип "Гибридная (клиент выбирает)"

## 📞 Поддержка

Если возникли вопросы или проблемы:
1. Проверьте настройки услуги в бизнес-админке
2. Убедитесь, что миграции базы данных выполнены
3. Проверьте консоль браузера на наличие ошибок
