# Инструкция по установке правил Cursor для проекта Ecme Marketplace

## Что такое .cursorrules?

`.cursorrules` - это файл с правилами и контекстом для AI-ассистента Cursor. Он помогает AI лучше понимать структуру проекта, стиль кода, архитектуру и лучшие практики, что позволяет получать более точные и релевантные ответы.

## Установка

### Шаг 1: Проверьте наличие файла

Файл `.cursorrules` уже создан в корне проекта. Убедитесь, что он находится по пути:

```
/Users/turbin/Desktop/REXTEN marketplace/ecme-admin/.cursorrules
```

### Шаг 2: Перезапустите Cursor

После создания или обновления файла `.cursorrules`:

1. **Закройте Cursor полностью** (Cmd+Q на Mac или полностью закройте приложение)
2. **Откройте Cursor снова**
3. **Откройте проект** заново

Cursor автоматически загрузит правила из `.cursorrules` при запуске.

### Шаг 3: Проверка работы

Чтобы убедиться, что правила загружены:

1. Откройте любой файл проекта
2. Используйте Cursor AI (Cmd+K или Cmd+L)
3. Задайте вопрос о проекте, например:
   - "Как работает авторизация в этом проекте?"
   - "Где находятся компоненты маркетплейса?"
   - "Как добавить новый API endpoint?"

AI должен отвечать с учетом контекста проекта и правил из `.cursorrules`.

## Обновление правил

Если вы обновили `.cursorrules`:

1. Сохраните файл
2. Перезапустите Cursor (полностью закройте и откройте снова)
3. Правила будут применены автоматически

## Использование с Cursor AI

Теперь при работе с Cursor AI:

### Примеры запросов:

1. **Создание компонента:**
   ```
   Создай компонент ServiceList для отображения списка услуг
   ```
   AI создаст компонент с учетом:
   - Структуры проекта
   - Использования React Query
   - Правильных импортов
   - Обработки ошибок

2. **Добавление API функции:**
   ```
   Добавь функцию для получения списка бронирований
   ```
   AI создаст функцию с учетом:
   - Структуры API в `src/lib/api/`
   - TypeScript типов
   - Обработки ошибок
   - Fallback на mock данные

3. **Создание защищенной страницы:**
   ```
   Создай страницу /business/reports для владельцев бизнеса
   ```
   AI создаст страницу с учетом:
   - Правильной структуры папок
   - ProtectedRoute компонента
   - Роли BUSINESS_OWNER
   - Layout и навигации

4. **Исправление ошибок:**
   ```
   Исправь ошибку авторизации на странице /client/profile
   ```
   AI будет знать:
   - Как работает авторизация
   - Где находятся stores
   - Как обрабатываются токены

## Решение проблем

### Правила не применяются

1. Убедитесь, что файл называется именно `.cursorrules` (с точкой в начале)
2. Убедитесь, что файл в корне проекта
3. Полностью перезапустите Cursor
4. Проверьте, что файл не пустой

### AI не следует правилам

1. Убедитесь, что правила актуальны
2. Обновите правила, если структура проекта изменилась
3. Перезапустите Cursor
4. В запросах к AI явно указывайте, что нужно следовать правилам проекта

### Нужно обновить правила

1. Откройте `.cursorrules`
2. Внесите изменения
3. Сохраните файл
4. Перезапустите Cursor
