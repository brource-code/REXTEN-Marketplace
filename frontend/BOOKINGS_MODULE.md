# Модуль бронирований - Документация

## Обзор

Модуль бронирований позволяет клиентам создавать бронирования, бизнесу управлять ими, а суперадмину видеть все бронирования платформы.

---

## 🔵 Для КЛИЕНТА (CLIENT)

### Доступные endpoints:
- `GET /api/client/bookings` - Получить список своих бронирований
- `POST /api/client/bookings/{id}/cancel` - Отменить бронирование

### Функционал:
1. **Просмотр своих бронирований**
   - Фильтрация по статусу (`status`)
   - Фильтрация предстоящие/прошедшие (`upcoming=true/false`)
   - Видит: название услуги, бизнес, дату, время, статус, цену, специалиста

2. **Отмена бронирования**
   - Может отменить только со статусом `pending` или `confirmed`
   - При отмене статус меняется на `cancelled`

### Frontend:
- Страница: `/client/booking`
- API функции: `src/lib/api/client.ts` - `getClientBookings()`, `cancelBooking()`

---

## 🟢 Для БИЗНЕСА (BUSINESS_OWNER)

### Доступные endpoints:
- `GET /api/business/bookings` - Получить все бронирования компании
- `GET /api/business/bookings/{id}` - Получить детали бронирования
- `POST /api/business/bookings` - Создать бронирование вручную
- `PUT /api/business/bookings/{id}` - Обновить бронирование
- `DELETE /api/business/bookings/{id}` - Удалить бронирование

### Функционал:
1. **Просмотр всех бронирований компании**
   - Фильтрация по статусу (`status`)
   - Фильтрация по дате (`date_from`, `date_to`)
   - Фильтрация по услуге (`service_id`)
   - Видит: клиента, услугу, специалиста, дату, время, статус, цену, заметки

2. **Создание бронирования**
   - Может создать бронирование вручную для клиента
   - Автоматическая проверка доступности слота
   - Можно указать специалиста, цену, статус

3. **Редактирование бронирования**
   - Может изменить дату, время, услугу, специалиста, цену, статус
   - При изменении даты/времени проверяется доступность
   - Может отменить бронирование (статус `cancelled`)

4. **Удаление бронирования**
   - Нельзя удалить завершенные бронирования (`completed`)

### Frontend:
- API функции: `src/lib/api/business.ts` - `getBusinessBookings()`, `createBusinessBooking()`, `updateBusinessBooking()`, `deleteBusinessBooking()`

## 🌐 Публичные endpoints (без авторизации)

### Доступные endpoints:
- `POST /api/bookings` - Создать бронирование (публично)
- `GET /api/bookings/available-slots` - Получить доступные слоты
- `POST /api/bookings/check-availability` - Проверить доступность слота

### Функционал:
1. **Создание бронирования**
   - Можно создать без авторизации
   - Требуется: `company_id`, `service_id`, `booking_date`, `booking_time`, `client_name`, `client_phone`
   - Автоматическая проверка доступности
   - Если пользователь авторизован, связывается с аккаунтом

2. **Получение доступных слотов**
   - Для выбранной даты, компании и услуги
   - Возвращает список свободных временных слотов

3. **Проверка доступности**
   - Проверяет конкретный слот на доступность

### Frontend:
- API функции: `src/lib/api/bookings.ts` - `createBooking()`, `getAvailableSlots()`, `checkSlotAvailability()`

---

## 🔧 BookingService - Логика бронирований

### Методы:
1. **`isSlotAvailable()`** - Проверка доступности слота
   - Проверяет пересечение временных интервалов
   - Учитывает существующие бронирования со статусами: `pending`, `confirmed`, `completed`

2. **`getAvailableSlots()`** - Получение доступных слотов
   - Генерирует список свободных временных слотов для даты
   - Учитывает рабочие часы и существующие бронирования

3. **`createBooking()`** - Создание бронирования
   - Проверяет доступность перед созданием
   - Автоматически получает цену из услуги
   - Создает запись в БД

## 📊 Структура данных Booking

```php
{
    id: number
    company_id: number
    user_id: number | null (может быть null для публичных бронирований)
    service_id: number
    specialist_id: number | null
    booking_date: date (Y-m-d)
    booking_time: time (H:i)
    duration_minutes: number (по умолчанию 60)
    price: decimal
    status: 'pending' | 'confirmed' | 'completed' | 'cancelled'
    notes: string | null (заметки бизнеса)
    client_notes: string | null (заметки клиента)
    cancelled_at: datetime | null
    cancellation_reason: string | null
}
```

## 🔐 Безопасность и изоляция

1. **Клиент** - видит только свои бронирования (`user_id = текущий пользователь`)
2. **Бизнес** - видит только бронирования своей компании (`company_id` через TenantMiddleware)
3. **Суперадмин** - должен видеть все (пока не реализовано)

---

## 📝 Статусы бронирований

- **`pending`** - Ожидает подтверждения (можно отменить)
- **`confirmed`** - Подтверждено (можно отменить)
- **`completed`** - Завершено (нельзя удалить)
- **`cancelled`** - Отменено
