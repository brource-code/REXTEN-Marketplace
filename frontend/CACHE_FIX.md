# Решение проблем с кешем Next.js

## Причины частых повреждений кеша

### 1. **`output: 'standalone'` в dev режиме**
- Проблема: `standalone` режим предназначен для production и может вызывать проблемы с кешем в dev
- Решение: Отключено для dev режима, используется только для production

### 2. **Агрессивная очистка кеша**
- Проблема: `onDemandEntries` с малыми значениями слишком быстро очищает страницы из памяти
- Решение: Увеличено `maxInactiveAge` до 120 секунд и `pagesBufferLength` до 10

### 3. **Множественные процессы Next.js**
- Проблема: Если запущено несколько процессов, они могут конфликтовать при записи в кеш
- Решение: Всегда проверяйте, что запущен только один процесс

### 4. **Проблемы с файловой системой**
- Проблема: На некоторых файловых системах (особенно сетевых) могут быть проблемы с атомарностью операций
- Решение: Убедитесь, что проект на локальном диске

## Команды для исправления

### Полная очистка и перезапуск:
```bash
# Остановить все процессы Next.js
pkill -f "next dev"

# Очистить все кеши
rm -rf .next .next/cache node_modules/.cache

# Перезапустить
npm run dev
```

### Проверка процессов:
```bash
# Проверить, сколько процессов Next.js запущено
ps aux | grep "next dev" | grep -v grep

# Должен быть только один процесс
```

### Мониторинг кеша:
```bash
# Проверить размер кеша
du -sh .next

# Проверить содержимое
ls -la .next/server | head -20
```

## Рекомендации

1. **Не используйте `output: 'standalone'` в dev режиме**
   - Это уже исправлено в `next.config.mjs`

2. **Увеличьте лимиты памяти, если нужно:**
   ```bash
   NODE_OPTIONS='--max-old-space-size=4096' npm run dev
   ```

3. **Используйте `.gitignore` для исключения кеша из git**
   - Это предотвратит конфликты при работе в команде

4. **Регулярно очищайте кеш при проблемах:**
   ```bash
   rm -rf .next/cache
   ```

5. **Проверяйте логи при ошибках:**
   ```bash
   tail -f /tmp/nextjs.log
   ```

## Если проблемы продолжаются

1. Проверьте версию Node.js (рекомендуется 18.x или 20.x)
2. Обновите зависимости: `npm update`
3. Переустановите зависимости: `rm -rf node_modules && npm install`
4. Проверьте права доступа к директории проекта
