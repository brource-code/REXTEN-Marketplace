# 🔧 Решение проблемы ERR_CONNECTION_REFUSED

## ✅ Что исправлено

1. Добавлен HTTP редирект (порт 80 → HTTPS 8443)
2. Nginx перезагружен
3. Все проверки пройдены

## 🌐 Как открыть в браузере

### Вариант 1: С портом (рекомендуется)
```
https://rexten.local:8443
```

### Вариант 2: Без порта (автоматический редирект)
```
http://rexten.local
```
Браузер автоматически перенаправит на `https://rexten.local:8443`

## ⚠️ Если браузер показывает предупреждение о сертификате

Это нормально для самоподписанного сертификата. Нужно:

1. **Chrome/Edge**: Нажмите "Дополнительно" → "Перейти на сайт rexten.local (небезопасно)"
2. **Firefox**: Нажмите "Дополнительно" → "Принять риск и продолжить"
3. **Safari**: Нажмите "Показать подробности" → "Посетить этот веб-сайт"

## 🔍 Проверка работы

Все работает через curl:
- ✅ `http://rexten.local` → редирект на HTTPS
- ✅ `https://rexten.local:8443` → HTTP 302

## 🐛 Если всё ещё не работает

1. **Очистите кэш браузера** (Cmd+Shift+R на Mac)
2. **Проверьте, что используете правильный URL**:
   - ✅ `http://rexten.local` (без порта)
   - ✅ `https://rexten.local:8443` (с портом)
   - ❌ `https://rexten.local` (без порта - НЕ РАБОТАЕТ)
3. **Проверьте, что серверы запущены**:
   ```bash
   ps aux | grep -E "nginx|php artisan|next dev" | grep -v grep
   ```
4. **Проверьте логи nginx**:
   ```bash
   tail -f /opt/homebrew/var/log/nginx/error.log
   ```
